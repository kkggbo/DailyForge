# DailyForge 本地基础设施启动说明

## 1. 目标

本文档用于说明如何通过 Docker Compose 在本地启动 DailyForge 开发所需的基础设施，并让 IDEA 中的后端项目能够直接连接到 MySQL 和 Redis。

当前本地基础设施包括：

- MySQL 8.0
- Redis 7
- phpMyAdmin

## 2. 文件说明

- [docker-compose.local.yml](</D:\Computer Science\DailyForge\deploy\docker-compose.local.yml>)
  - 本地开发用 Docker Compose 配置
- [local-services.example.md](</D:\Computer Science\DailyForge\deploy\local-services.example.md>)
  - 固定本地连接参数说明

## 3. 后端配置说明

后端当前已改为固定本地开发配置，不依赖环境变量。

默认连接参数如下：

- Spring Profile：`dev`
- MySQL Host：`localhost`
- MySQL Port：`3306`
- MySQL Database：`dailyforge_dev`
- MySQL Username：`dailyforge`
- MySQL Password：`change_me_app`
- Redis Host：`localhost`
- Redis Port：`6379`
- Redis Database：`1`

## 4. 启动步骤

1. 进入 [deploy](/D:\Computer Science\DailyForge\deploy) 目录
2. 执行以下命令启动容器：

```powershell
docker compose -f docker-compose.local.yml up -d
```

3. 确认容器状态：

```powershell
docker compose -f docker-compose.local.yml ps
```

4. 启动后端项目

## 5. 访问地址

- MySQL
  - `localhost:3306`
- Redis
  - `localhost:6379`
- phpMyAdmin
  - `http://localhost:8081`

## 6. IDEA 运行建议

如果你直接在 IDEA 中运行后端，当前无需额外补环境变量，只要本地 Docker 服务已经按本文档启动即可。

## 7. 当前方案的定位

这套 Compose 配置是本地开发方案，目标是：

- 快速启动依赖环境
- 让后端能够连接本地 MySQL 和 Redis
- 让 IDEA 中的后端应用可直接连接

它不是最终生产部署方案。后续上云时，应再单独准备：

- `docker-compose.prod.yml`
- 更严格的密码与网络隔离配置
- 备份和监控方案
